# President Display-Name Helper Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a shared, pure-TypeScript utility module at `src/lib/president-names.ts` that turns raw administration data (president name, term number, whether the dataset contains more than one term for that president) into a formatted display string, plus a higher-level helper that walks a pardon-details content collection and returns a per-slug index of display names and counts. This is the single source of truth that the headline on the home page (#7), the president filter pills on the search page (#6), and the comparison view (#8) will all use.

**Architecture:** Two exported functions in one module, fully covered by vitest unit tests with inline fixture data — no database access, no content-collection reads. The pure formatter `formatAdministrationDisplayName` is a one-liner that either returns the president's name (single-term case) or suffixes it with an ordinal term label (`"First Term"`, `"Second Term"`, etc.). The index builder `getAdministrationIndex` walks a collection in two passes: first to count how many distinct term numbers each `president_name` has, then to produce a `Map<administration_slug, AdministrationIndexEntry>` whose display-name field uses the formatter with the single-term-or-not bit filled in. Consumers import `getAdministrationIndex(entries)` and get everything they need keyed by slug.

**Tech Stack:** TypeScript, vitest 4, oxlint, oxfmt. No runtime dependencies beyond what the project already uses.

**Related issues:** Closes #10. Consumed by #6, #7, #8.

**Sibling plan:** `docs/superpowers/plans/2026-04-10-unclamp-content-collection.md` (issues #9 and #11). Fully independent — this plan can land before, after, or in parallel with the unclamp. It does not need the content collection to contain multi-administration data to be testable, because the tests use inline fixtures rather than `getCollection`.

## Context

GitHub issue #10 identified that #6, #7, and #8 all need to format an administration slug like `biden-1` or `trump-2` into a human-readable label (`"Joe Biden"` or `"Donald Trump (Second Term)"`). Without a shared helper, each of the three feature issues ends up reimplementing the same "is this president's only term in the dataset? what ordinal do I suffix?" logic, which is a classic DRY violation waiting to happen.

The pardon-details loader at `src/loaders/pardon-details.ts:9-37` already carries `president_name`, `term_number`, and `administration_slug` on every entry (selected from a join against the `administrations` table at lines 79, 92-93). That means the helper can work entirely off collection entries, with no DB access and no need to re-query for metadata.

The "is this president's only term" decision is scoped to whatever data the caller hands in. If the current dataset only has `biden-1` for Biden, then Biden is treated as single-term and rendered without a suffix. If `biden-2` later appears in the data, the same Biden entries would then render as `"Joe Biden (First Term)"` automatically — no code changes. This is intentional: the display reflects what's in the collection, not historical reality.

### Deviation from issue #10's suggested signature

Issue #10 sketched the helper as `getAdministrationDisplayName(slug: string): string` — a single slug-keyed function. This plan instead ships two functions: a pure `formatAdministrationDisplayName(presidentName, termNumber, isOnlyTerm)` that takes raw inputs and returns a string, plus a `getAdministrationIndex(entries)` walker that takes a collection and returns a `Map<slug, AdministrationIndexEntry>`. Consumers who want "slug → display name" write `getAdministrationIndex(entries).get(slug)?.displayName`.

The reason for the change: a slug-only signature implies the helper has to hold state (either a cached lookup from some loader, or a static JSON snapshot generated at build time), which makes it awkward to unit test and introduces a hidden dependency on the content collection loader's behavior. The two-function approach is purely functional — each function takes everything it needs as arguments — which means the tests run with inline fixtures and have no file-system or Astro-loader coupling. Consumers already hold a `getCollection("pardonDetails")` result in their page frontmatter (see `src/pages/index.astro:26`, `src/pages/search.astro:11`, etc.), so passing it to an index builder is a natural call-site extension rather than a new data-layer touchpoint.

The issue #10 body itself acknowledged this tradeoff: "The helper needs access to the administrations table (either at build time via the content collection loader pattern, or a static JSON snapshot generated by the scraper — whichever matches existing conventions)." This plan picks the content-collection pattern, which matches how the rest of `src/lib/pardon-stats.ts` works (every helper there also takes a `CollectionEntry[]` argument rather than reading from global state).

/**
 * Format an administration's display name for UI use.
 *
 * Single-term presidents (isOnlyTerm=true) render as just the name.
 * Multi-term presidents render with an ordinal suffix, e.g.
 * "Donald Trump (First Term)".
 */
export function formatAdministrationDisplayName(
  presidentName: string,
  termNumber: number,
  isOnlyTerm: boolean,
): string {
  if (isOnlyTerm) {
    return presidentName;
  }
  return `${presidentName} (${ordinalTermLabel(termNumber)})`;
}

const ORDINAL_TERM_LABELS: Record<number, string> = {
  1: "First Term",
  2: "Second Term",
  3: "Third Term",
  4: "Fourth Term",
};

function ordinalTermLabel(termNumber: number): string {
  return ORDINAL_TERM_LABELS[termNumber] ?? `Term ${termNumber}`;
}
```

/**
 * A single administration's row in the index — everything a filter UI,
 * headline, or comparison column needs, keyed by administration slug.
 */
export interface AdministrationIndexEntry {
  slug: string;
  displayName: string;
  presidentName: string;
  termNumber: number;
  count: number;
}

/**
 * Minimum shape the index builder needs from each collection entry.
 * Matches what the pardon-details loader already exposes via its schema
 * (administration_slug, president_name, term_number are joined from the
 * administrations table during loading).
 */
interface MinimalEntry {
  data: {
    administration_slug: string;
    president_name: string;
    term_number: number;
  };
}

/**
 * Walk a pardon-details collection and build a per-slug index of
 * display metadata. Handles the "is this president's only term in the
 * dataset" decision by counting distinct term numbers per president_name
 * in a first pass, then producing the index in a second pass.
 *
 * Usage in an Astro page:
 * ```ts
 * import { getCollection } from "astro:content";
 * import { getAdministrationIndex } from "../lib/president-names";
 *
 * const entries = await getCollection("pardonDetails");
 * const index = getAdministrationIndex(entries);
 * // index.get("biden-1")?.displayName === "Joe Biden"
 * ```
 */
export function getAdministrationIndex(
  entries: MinimalEntry[],
): Map<string, AdministrationIndexEntry> {
  const termsPerPresident = new Map<string, Set<number>>();
  for (const entry of entries) {
    const { president_name, term_number } = entry.data;
    let terms = termsPerPresident.get(president_name);
    if (!terms) {
      terms = new Set();
      termsPerPresident.set(president_name, terms);
    }
    terms.add(term_number);
  }

  const index = new Map<string, AdministrationIndexEntry>();
  for (const entry of entries) {
    const { administration_slug, president_name, term_number } = entry.data;
    const existing = index.get(administration_slug);
    if (existing) {
      existing.count++;
      continue;
    }
    const isOnlyTerm = termsPerPresident.get(president_name)!.size === 1;
    index.set(administration_slug, {
      slug: administration_slug,
      displayName: formatAdministrationDisplayName(
        president_name,
        term_number,
        isOnlyTerm,
      ),
      presidentName: president_name,
      termNumber: term_number,
      count: 1,
    });
  }

  return index;
}
````
